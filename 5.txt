1 你修改了一个代码文件，想让外地工作的同事马上使用，请问需要几个步骤，用哪些命令 （10分）
(1)切换到从库中克隆下来的版本库(.git)所在路径
	cd 路径
(2)与同事共用的版本库进行连接（已有连接不新建连接git remote -v可看）
	git remote add 连接名 远程库路径（ssh/http） 
(2)添加进此分支下的暂存区
	git add 文件名
(3)提交，写说明
	git commit -m "说明"
(4)推送到远程库
	git push 连接名 分支名
(5)若发生代码冲突，先处理冲突再推送
	git pull
	


2 你已经在本地编写了若干的代码，还没有提交到本地库，这个时候有其他同事通知你他的代码更新了，需要你马上跟新，你要怎么做几个步骤，用哪些命令？（15分）
(1)从远程库上把最新版本的代码克隆下来
	git clone 远程库地址
(2)将本地库与最新版本的分支合并
	git merge 新版本分支
(3)将本地编写的代码加进暂存区
	git add 文件名
(4)提交，写说明
	git commit -m "说明"
	
	
3 除了命令行外，你所学习了的git 图形化工具有哪些？它的优缺点？（10分）
(1)工具：GIT GUI 、github for windows 、像Intellij IDEA等IDE
(2)优点：可视化操作，简单不需记命令，效率高
	缺点：相对命令行来说，不能记录每一步操作


4 每次提交写注释时应该包含哪些内容？为什么要写注释（10分）
内容
(1)对已进行的新的操作进行说明，包括增删改等操作
(2)有需要可加版本号
(3)

原因
(1)对本次的修改进行记录，让自己或其它协同工作的同事清楚本次改动的内容
(2)方便版本恢复
(3)方便查看时间线及分支合并
